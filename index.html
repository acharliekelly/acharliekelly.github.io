Resumption

Full-stack resume management application

During the resume-writing process at GA, I found that I really had too many different types of experience to fit easily on a single-page, general purpose resume. There are commercial applications that let you create multiple resumes and host them online, but they generally require a fee; besides, they offered a lot of features I didn't need, while failing to offer some things I wanted.

So my full-stack app lets me save multiple versions of each resume section (ie Experience, Skills, etc). Each resume I create can combine multiple sections, and apply one of several templates. I can then publish the resume to a publicly-accessible URL. Then I can provide a specific URL to each job I'm applying for. If a role doesn't exactly match any of my existing resumes, I can easily create a new one just by recombining existing sections.

The back-end is a Rails API based on 4 relational tables, including a many-to-many join of sections to layouts.
